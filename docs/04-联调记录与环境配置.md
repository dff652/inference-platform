## 最后更新：2026-03-16

## 一、环境总览

本项目涉及 **3 个 conda 环境**，各自承担不同职责：

| 环境 | Python | 用途 | 关键依赖 |
|------|--------|------|----------|
| `inference-platform` | 3.11 | 平台后端 + Celery worker | FastAPI, SQLAlchemy, Celery |
| `chatts` | 3.12 | GPU 推理（ChatTS / Qwen） | transformers 5.3, bitsandbytes, torch |
| `ts` | 3.8 | CPU 推理（ADTK / Ensemble 等） | adtk, statsmodels, kneed, pyod |

执行适配器根据推理方法自动切换 Python 解释器（`GPU_METHODS = {"chatts", "qwen"}`）。

## 四、各方法环境与验证状态

| 方法 | conda 环境 | GPU | 量化 | 显存 | 状态 |
|------|-----------|-----|------|------|------|
| adtk_hbos | ts (3.8) | 不需要 | - | - | ✅ 已验证 |
| chatts | chatts (3.12) | cuda:0/1 | 4-bit (force) | ~7.6GB | ✅ 已验证 |
| qwen | chatts (3.12) | cuda:1 | bf16 (8B) / 4-bit (27B) | ~18GB / 需48GB+ | ✅ 8B 已验证 |
| ensemble | ts (3.8) | 视方法组合 | - | - | 待验证 |
| wavelet | ts (3.8) | 不需要 | - | - | 待验证 |
| isolation_forest | ts (3.8) | 不需要 | - | - | 待验证 |
| timer | 待确认 | cuda:0 | - | - | 待验证 |

### load_in_4bit 参数说明

| 值 | 含义 |
|-----|------|
| `auto` | 默认。ChatTS-8B 自动禁用 4-bit（质量优先），其他模型启用 |
| `true` | 强制启用 4-bit（但 ChatTS-8B 仍会被自动禁用） |
| `false` | 不使用量化，加载 bf16 全精度 |
| `force` | 强制 4-bit，即使是 ChatTS-8B 也不禁用（显存不足时使用） |

- 状态：代码已支持（多卡 + 4-bit），需 ≥48GB 显存设备

## 七、输出目录结构约定

```
{data_path}/
  └── {task_name}/          # 默认 "global"
      └── {method}/         # 如 "adtk_hbos", "chatts", "qwen"
          ├── {method}_{downsampler}_{point}_{date}.csv
          ├── {method}_{downsampler}_{point}_{date}_metrics.json
          └── {method}_{downsampler}_{point}_{date}_segments.json
```

### metrics.json 格式

```json
{
  "version": 1,
  "summary": {
    "score_avg": 69.98,
    "score_max": 69.98,
    "segment_count": 1
  },
  "method": "adtk_hbos",
  "point_name": "PI_20412.PV",
  "task_id": "999"
}
```

### segments.json 格式

```json
[
  {
    "start": 2696,
    "end": 2705,
    "score": 69.98,
    "length": 10
  }
]
```
