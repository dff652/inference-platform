## 四、发现的问题与修复

| 问题 | 原因 | 修复 |
|------|------|------|
| 后端启动失败 sqlite3.OperationalError | DATABASE_URL 使用相对路径 ./data/ | 改为基于 PROJECT_ROOT 的绝对路径 |
| /stats 接口 500 | TaskStatusStats 缺少 draft 字段 | 添加 draft: int = 0 |
| ts 环境缺少多个包 | 旧项目可能在其他机器上运行过，当前机器环境不完整 | 补装 7 个包 |

## 五、输出目录结构约定

旧项目 run.py 的输出目录结构：

```
{data_path}/
  └── {task_name}/          # 默认 "global"
      └── {method}/         # 如 "adtk_hbos"
          ├── {method}_{downsampler}_{point}_{date}.csv
          ├── {method}_{downsampler}_{point}_{date}_metrics.json
          └── {method}_{downsampler}_{point}_{date}_segments.json
```

适配器结果收集逻辑：扫描 output_dir 下所有 .json 文件，解析 annotations/segments。
