# 联调记录与环境配置

## 最后更新

2026-03-16

## 1. 文档定位

这份文档是当前项目的环境安装基线，负责说明：

- 宿主机前提条件
- 3 套 conda 环境的安装方式
- 路径和环境变量要求
- 模型目录和旧项目依赖
- 已验证的联调记录

说明：

- 截至 2026-03-16，GPU 推理实际运行在 `transformers` 常驻服务上
- 平台侧使用 OpenAI 兼容 HTTP 接口调用这些服务
- 真正切换到 vLLM 仍在后续规划中

## 2. 宿主机前提

必需：

- Linux
- Conda
- Redis
- Node.js 18+
- Python 3.11 / 3.12 / 3.8 可创建 conda 环境

GPU 方法额外需要：

- NVIDIA 驱动
- CUDA 环境
- 可访问的模型权重目录

## 3. conda 环境

### 3.1 平台后端环境 `inference-platform`

```bash
conda create -n inference-platform python=3.11 -y
conda activate inference-platform
cd /home/dff652/dff_project/inference-platform
pip install -r backend/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

用途：

- FastAPI
- Celery worker
- 平台内 CPU 调度代码
- 脚本与种子数据

### 3.2 GPU 环境 `chatts`

```bash

conda activate chatts
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
pip install 'transformers>=5.0' accelerate bitsandbytes

    tsdownsample PyWavelets psutil pynvml ruptures seaborn \
```

## 6. 启动方式

### 6.1 手工联调

```bash
# Redis
redis-server --daemonize yes

# GPU 服务
cd /home/dff652/dff_project/inference-platform
./services/start_vllm.sh all

# API
cd /home/dff652/dff_project/inference-platform/backend
/home/dff652/miniconda3/envs/inference-platform/bin/uvicorn app.main:app --host 0.0.0.0 --port 8100

# Celery
cd /home/dff652/dff_project/inference-platform/backend
/home/dff652/miniconda3/envs/inference-platform/bin/celery -A app.core.celery_app worker --loglevel=info

# Frontend
cd /home/dff652/dff_project/inference-platform/frontend
npm run dev
```

### 6.2 Docker + 宿主机 GPU 服务

```bash
cd /home/dff652/dff_project/inference-platform
cp .env.example .env
docker compose up -d
```

注意：

- `docker-compose.yml` 不会自动启动 ChatTS/Qwen 服务
- GPU 服务要在宿主机额外启动
- 容器内 `VLLM_*_ENDPOINT` 必须能访问到宿主机 GPU 服务
